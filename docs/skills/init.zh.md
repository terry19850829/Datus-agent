# Init

`init` 是初始化项目工作区的轻量方式。它扫描你的工作目录和已配置的数据源，然后写下一份简洁、持久的项目概览，让后续会话一开始就带有上下文——而不构建较重的向量知识库。

在 REPL 中使用 `/init` 命令运行。

## 功能作用

- 扫描工作目录与已配置的数据源（名称、类型、表数量）。
- 在项目根目录写一份 `AGENTS.md` 清单：架构、目录地图、服务、数据资产，以及知识索引。
- 把持久的业务事实存为基于文件的 knowledge，把与 agent 绑定的偏好存为 memory。

它属于**轻量**层级：快（数秒到一分钟）、不建向量索引、无确认门。需要向量索引的知识库时，请使用 [`/build-kb`](build_kb.md)。

## 何时使用

- 新项目工作区还没有 `AGENTS.md`。
- 你明确想要初始化或重新扫描项目。
- 文件或表结构有较大变化，清单已过时。

当一份最新的 `AGENTS.md` 已存在且没有实质变化时，可以跳过它。

## 如何使用

```text
/init
```

你可以在命令后附加自由文本提示——一个目标、一个范围，或要重点关注的文件 / 表：

```text
/init focus on the sales and finance schemas
```

不带提示时，`init` 会进行广泛扫描。重新运行会刷新清单，且不会重复写入 knowledge 条目。

## Init 与 Build KB 的区别 { #init-vs-build-kb }

| | `init` | [`/build-kb`](build_kb.md) |
|---|---|---|
| 速度 | 快（数秒到一分钟） | 较慢（数分钟） |
| 产出 | `AGENTS.md` + knowledge / memory | 向量知识库：语义模型、指标、参考 SQL |
| 确认 | 无 | 生成清单确认门 |
| 向量索引 | 否 | 是 |
| 时机 | 最先、总是先做 | init 之后，当你需要语义检索时 |

典型流程：先运行 `/init` 获得即时上下文，需要向量索引知识库时再运行 `/build-kb`。

## 注意事项

- knowledge 以文件形式存放在 `knowledge/` 下；memory 与 agent 绑定并跨会话保留。
- `init` 只做清单盘点——不执行 SQL 查询。
- 生成语义模型、指标和参考 SQL 是 [`/build-kb`](build_kb.md) 的职责。
