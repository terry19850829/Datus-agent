# Session Summarize

`session-summarize` 回顾一次聊天会话并持久化重要内容。它会通读对话，识别值得长期保留的产出，并把每一项路由到合适的存储。在一次工作会话结束时使用它，让下一次能从你离开的地方继续。

## 功能作用

一次工作会话往往会产生值得保留的内容——你发现的一条业务规则、你陈述的一项偏好、你验证过的一段查询。`session-summarize` 会回顾整段对话，挑出这些持久的产出，并把每一项保存到合适的存储（通过 [Storage Classify](storage_classify.md)）：事实存入 knowledge，偏好存入 memory，已验证查询存入参考 SQL，依此类推。

## 何时使用

- 会话产生了持久的事实、偏好或已验证的 SQL。
- 你想保存或记住这次会话。
- 在切换到其他项目或任务之前。

## 如何使用

在会话结束时以 skill 方式运行 `session-summarize`。你可以选择性地给它聚焦提示；不带提示时，它会回顾整段会话。它只捕获值得保留的产出——一次性的来回讨论会被略去。

## 示例

你用一次会话弄清了一个流失率查询。过程中你确认了一段已验证的 SQL，得知 `plan_tier = 0` 表示免费账户，并决定更喜欢把结果四舍五入到整数百分比。结束时，`session-summarize` 会保存：

- 流失率查询 → **参考 SQL**，
- `plan_tier = 0` 规则 → **knowledge**，
- 四舍五入偏好 → **memory**。

下一次会话会自动获取这三者。
