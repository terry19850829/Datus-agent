# Memory Organization

`memory-organization` 审计并整理项目的持久化存储——knowledge、memory、语义模型，以及 `AGENTS.md` 索引。它会发现重复、矛盾、错放和缺失的条目，然后提出修复方案，并应用你批准的那些。

定期运行它，或在存储变得杂乱时运行。

## 功能作用

随着时间推移，knowledge 和 memory 会不断累积：同一条事实被记录两次、两条条目开始冲突、本应属于 knowledge 的内容被存成了 memory，或者一条明显的事实始终没被写下。`memory-organization` 把这些存储作为整体审视，让它们回到干净、一致的状态。

## 它会检查什么

- **重复**——同一条事实被记录在多个地方。
- **矛盾**——彼此冲突的条目。
- **错放**——本该是 memory 的 knowledge，反之亦然。
- **缺口**——本应记录却没有的明显事实。
- **过期**——不再成立的条目。

## 何时使用

- 存储变得杂乱或自相矛盾。
- 大量使用后新增了许多条目。
- knowledge 与 memory 发生漂移或开始重叠。
- 你想做一次清理或重新整理。

## 如何使用

在 REPL 中以 skill 方式运行 `memory-organization`。你可以选择性地告诉它聚焦哪些存储；不带提示时，它会审计全部。它会汇报发现的问题并提出修复方案，在应用前先经你批准——任何改动都在你的掌控之中。

## 示例

经过几周的高强度使用，你的项目里同一条“财年从四月开始”的规则被写进了两个 `knowledge` 文件，一条 `memory` 备注与更新的偏好相矛盾，还有一条过期条目指向你已删除的表。运行 `memory-organization` 会标记出这三处，提出合并重复项、移除过期条目、调和冲突——并在你确认后应用。
